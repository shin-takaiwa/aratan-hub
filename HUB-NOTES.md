# hub — 公開アプリのハブ

唯一「育て続ける」アプリ。週アプリ（使い捨て前提）とは物理的に分ける。

## 役割

- 公開した週アプリの **リンク集**
- 各アプリの **メタ情報**（週・ピッチ・ターゲット・状態）
- **計測の集約**（Vercel Analytics / PostHog 等のダッシュボードへの導線）

> アプリ本体のDBに利用計測を溜めない。ハブはあくまで集約と入口。

## カード登録のルール

- アイコンは **絵文字フォールバックに頼らず、必ず実アイコン（`icon`）を設定する**。
- 実アイコンは各アプリの favicon（`src/app/icon.svg` 等）を `public/<slug>.svg` にコピーして使う＝タブとハブで同じ顔にする。
- `emoji` / `accent` は icon が無い場合の暫定表示。登録時の置き忘れ防止のため埋めておくが、公開状態で emoji タイルのままにしない。

## git / デプロイ

- ここは **独立リポ・独立 Vercel プロジェクト**（基盤リポの `.gitignore` で除外済み）。
- 最初に作るときは `template` のコピーでも、専用に作ってもよい。育てる前提で構わない。

```bash
cd hub
git init
# Next.js 等で立ち上げ、Vercel に独立プロジェクトとして接続
```
